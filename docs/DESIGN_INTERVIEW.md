# Ouroboros 전문가 인터뷰 기반 요구사항

이 문서는 Q00/ouroboros의 Socratic interview 원칙(범위·비목표·출력·검증·운영 경계를 닫은 뒤 구현)을 적용한 설계 기록이다. 원래 기획 문서와 공개 자료에서 사실을 먼저 확인하고, 포트폴리오라는 명시된 목적 안에서 실행을 바꾸는 질문에 답했다.

## 전문가 질문과 답변

### Q1. 이것은 상용 IPS인가, 면접에서 직접 실행하고 설명할 수 있는 NIDS 포트폴리오인가?

**답변:** 후자다. 원시 패킷 차단, 커널 드라이버, 분산 수집기는 범위 밖이다. 입력 단위는 네트워크 flow이며, 로컬에서 권한 없이 재현 가능한 탐지·저장·시각화·조사 흐름이 완료 조건이다.

### Q2. 지도학습 정확도 하나를 앞세우는 것이 타당한가?

**답변:** 아니다. NSL-KDD는 비교 가능한 고전 벤치마크지만 오래된 트래픽이고, CICIDS2017도 특정 실험 환경 데이터다. 임의 행 분할은 같은 호스트/시간대 패턴을 양쪽에 섞어 누수를 만들 수 있다. 이 구현은 허위 정확도 수치를 표시하지 않고, 정상 기준 이상 탐지와 설명 가능한 규칙을 제공한다. 실데이터 평가는 날짜/호스트 단위로 분리해야 한다.

### Q3. 탐지 결과를 운영자가 신뢰하려면 무엇을 보여줘야 하는가?

**답변:** 이진 라벨만으로는 부족하다. 공격 유형, 위험 점수, 이상 점수, 신뢰도, 일치한 규칙 ID, 사람이 읽을 수 있는 판단 근거, 모델 버전을 이벤트와 함께 저장한다.

### Q4. 알려진 공격과 알려지지 않은 이상을 어떻게 함께 다루는가?

**답변:** Port Scan·Brute Force·DoS·대량 유출·C2 포트는 규칙으로 높은 설명력을 확보한다. 규칙에 없는 흐름 형태는 Isolation Forest로 정상 기준과의 거리를 측정한다. 두 신호는 확률의 합집합 방식으로 결합하되, 이를 보정된 공격 확률이라고 주장하지 않는다.

### Q5. “실시간”의 계약은 무엇인가?

**답변:** 데모 생성기 또는 POST API로 들어온 flow를 동기 탐지·저장하고 WebSocket으로 연결된 화면에 전달한다. 연결이 끊기면 UI가 재접속하며, 영속 저장소가 이벤트 이력을 복구한다. 메시지 브로커 기반 exactly-once 전달은 비목표다.

### Q6. 데이터셋은 어떻게 연결하되 저장소를 무겁게 만들지 않는가?

**답변:** 원본은 포함하지 않는다. `scripts/train.py --csv`가 CICIDS2017 CSV의 BENIGN flow를 읽어 기준 모델을 만든다. 합성 정상 기준은 설치 직후 실행되는 smoke/demo용이며 실성능 근거가 아니다.

### Q7. 성공 기준은 무엇인가?

**답변:** (1) 잘못된 IP/포트는 API에서 거부, (2) 다섯 규칙 공격이 재현 가능하게 검출, (3) 정상 예시는 경보가 아님, (4) 이벤트 상태가 저장, (5) 집계가 이벤트와 일치, (6) 실시간 화면이 갱신, (7) 테스트와 컴파일이 통과하는 것이다.

### Q8. 보안상 실패 모드는 무엇인가?

**답변:** 입력 크기는 Pydantic 경계로 제한하고, SQL은 파라미터 바인딩을 사용하며, UI는 모든 네트워크 문자열을 escape한다. 이벤트 큐가 느린 구독자로 포화되면 해당 큐를 제거해 수집 루프를 보호한다. 인증·TLS·멀티테넌시는 로컬 포트폴리오 범위 밖이며 외부 공개 전 반드시 추가해야 한다.

## Restate gate

> **goal:** 공개 flow 데이터로 확장 가능하면서도 로컬에서 즉시 재현되는 설명 가능한 하이브리드 NIDS를 구현하고, 탐지 근거·이력·실시간 SOC 조사 흐름과 자동 검증을 함께 제공한다.

이 한 문장이 원 기획의 “AI 기반 침입탐지 + 실시간 모니터링 + 면접 설명 가능성”을 실행 가능한 경계로 보존한다.

## 교차 검증 자료

- [UNB CICIDS2017](https://www.unb.ca/cic/datasets/ids-2017.html): PCAP과 CICFlowMeter 기반 라벨 flow CSV, 정상/공격 트래픽 구성.
- [UNB NSL-KDD](https://www.unb.ca/cic/datasets/nsl.html): KDD 중복 레코드 문제를 줄여 비교 가능성을 개선한 고전 벤치마크.
- [NIST SP 800-94](https://csrc.nist.gov/pubs/sp/800/94/final): IDPS 설계·구현·운영·유지관리 가이드.
- [scikit-learn Isolation Forest](https://scikit-learn.org/stable/modules/outlier_detection.html#isolation-forest): 고차원 이상치를 무작위 분할로 격리하는 모델의 공식 설명.
- [FastAPI WebSockets](https://fastapi.tiangolo.com/advanced/websockets/): 실시간 이벤트 전달 구현 계약.
